### 問題点

1. **用語集に「64 Pad Explorer」の独立項目がない**
   - Push, Launchpad, Moveにはそれぞれ項目がある
   - 64PEはうりなみさん自作ツールなのに、用語集で説明されていない

2. **FAQ Q2「音楽理論がわかりません」に64PEの言及がない**
   - PadManと川三64パッドは紹介されているが、64PEは未記載
   - 64PEの方が機能的に上位（スケール31種、テンション、ボイシング等）

3. **GA4トラッキングは設定済み**
   - `outbound_click` + `link_destination: '64pad'` で計測可能
   - ただしクリック元データ（どの用語/FAQ/記事から遷移したか）は要link_destination有効化

## 提案（うりなみさん確認事項あり）

### A. 用語集に「64 Pad Explorer」項目を追加

**追加先**: `bot_data/HPS用語集.md` → `sync_data.py` → `glossary.json`

```
title: 64 Pad Explorer (64パッドエクスプローラー)
category: ツール
description: うりなみが開発した64パッド向けの学習ツール。
             スケール・コード・ボイシングを64パッドグリッド上に
             可視化する。ブラウザで無料で使える。
hps_practice: 31スケール、テンション拡張コード、ボイシング候補の
              表示に対応。HPS会員にはTASTYボイシング・ストック
              ボイシング機能を開放。
              📚 [64 Pad Explorer](https://murinaikurashi.com/apps/64-pad/)
```

**⚠️ うりなみさん確認**: 用語集にうりなみさん自作ツールを載せる方針でOKか

### B. FAQ Q2に64PEを追加

現在のQ2「音楽理論が全くわかりません」の学習支援ツール欄に追加:

```
- [64 Pad Explorer](https://murinaikurashi.com/apps/64-pad/)
  （うりなみ開発）：スケール・コード・ボイシングの64パッド可視化
```

**理由**: PadManと川三は外部開発。自作ツールが紹介されていないのは不自然

### C. ブログ記事からの導線（#811と連携）

blog.jsonに64PEへのリンクが7件あるが、直接的な「64 Pad Explorerの使い方」記事がブログにない。

**提案**: うりなみさんが64PE紹介記事をブログに書くか、guide.html自体を導線として活用
→ #811（ランディングページコピー調整）と連携

### D. Discord bot応答への組み込み

Discord botでスケール・コード関連の質問が来た際に、64PEへのリンクを応答に含める。

**現状**: `HPS用語集.md` に項目がないため、botが64PEを案内できない

