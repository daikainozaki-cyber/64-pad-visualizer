## 問題点と対策

### 1. Desktop/Plugin版の同期

**現状**: Web版のコードを手動で`sync-webui.sh`でDesktop/Pluginにコピー。
**問題**: 同期漏れ、ビルド違い（CLAUDE.mdに複数のインシデント記録あり）。
**対策**: Web/Desktop/Pluginは別リポジトリ（CLAUDE.md防御ルール準拠）。pad-coreのsubmodule更新で共通ロジックを同期。アプリ固有コードは各リポジトリに閉じる。

### 2. キャッシュ無効化

**現状**: XserverはCloudflare CDN。sw.jsが7日キャッシュ。バージョンバンプ漏れが4回以上。
**対策**:
- `.htaccess`でsw.jsにno-cacheヘッダー
- SW登録URLに`?v=`パラメータ
- デプロイスクリプトにバージョンバンプチェックを内蔵

### 3. CI競合

**現状**: 連続pushでGitHub ActionsのCI結果が上書き。
**対策**: `concurrency`設定で同時実行を防ぐ。

### 4. submodule更新手順

```bash
# pad-coreを更新
cd pad-core
git pull origin main
cd ..
git add pad-core
git commit -m "Update pad-core to latest"
git push
```

**注意**: pad-coreの変更は3リポジトリ全てで`git submodule update`が必要。

## 200pad対応（#895）

200padレイアウトはpad-coreの`data.js`にレイアウト定義を追加するだけ。新リポジトリは不要。

```javascript
// data.js に追加
const LAYOUT_200PAD = {
  rows: 14, cols: 14,  // or appropriate layout
  interval: 5,         // fourths-based
  // ...
};
```

64-pad-visualizerの描画は`render.js`の`ROW_INTERVAL`パラメータ化（#697）で対応。
