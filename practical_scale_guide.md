# Practical Scale Selection Guide（実践的スケール選択ガイド）

**作成**: 2026-02-16
**出典**: うりなみさんの実践知 + 『HOW TO IMPROVISE』『ザ・ジャズ・セオリー』『ツー・ファイブ・ジャズ・ライン』『HOW TO COMP』
**用途**: AI用リファレンス（64 Pad ExplorerのAvailable Scaleソート実装用）

---

## なぜこのファイルが必要か

AIの学習データは音楽理論を「教科書的に正しく」知っている。しかし**実践での重み付け**を知らない。

| AIが知っていること | AIが知らなかったこと |
|---|---|
| iii7にはPhrygianが対応する | 実践ではDorianを使う |
| 7つのモードは平等に存在する | HMモードは2つしか実戦で使わない |
| Alteredスケールの構成音 | アボイド0だからHMP5↓より優先される |
| セカンダリードミナントの理論 | 解決先の2択で全部決まる |

**このファイルは「重み」を記録する。理論と実践の差分そのもの。**

---

## ジャンルによる重みの違い（重要な前提）

**このガイドの実践的序列はジャズ文脈が中心。** ジャンルが変われば重みも変わる。

| ジャンル | マイナー系の傾向 | HMモードの扱い |
|---|---|---|
| **ジャズ** | ドリアン一発。HMはHMP5↓とFunc.Dimだけ | 他のHMモードは「理論的」 |
| **ネオソウル** | Im7でHarmonic Minor普通に弾く | HMの「エキゾチックな暗さ」がむしろ求められる |
| **クラブ/ハウス** | ドリアン・フリジアン多用 | 文脈次第 |
| **ラテン/フラメンコ** | フリジアン・HMP5↓ | Phrygian Dominantが主役 |

**「HMモードは2つだけ実戦的」はジャズの常識。ネオソウルでは通用しない。**

書籍も古いものはジャズ偏重。現代音楽での使われ方は変化している。

## 大原則（2つだけ）

### 原則1: アボイド=0のスケールを選べ

アボイドノート（コードトーンと半音でぶつかる音）が少ないスケールほど実践的。
ジャズミュージシャンがスケールを選ぶ最大の基準。

### 原則2: 解決先がメジャーかマイナーかで決まる

ドミナント7th系のスケール選択は、そのコードが**何に解決するか**で2択に収束する。
コード単体では決まらない。流れを見る。

## 1. ダイアトニックコードの実践マップ

| 機能 | 理論（教科書） | 実践（ジャズ） | 理由 |
|---|---|---|---|
| **I△7** | Ionian | **Lydian** | Ionianの4th(11th)がアボイド。Lydianは全音使える |
| **ii7** | Dorian | Dorian | アボイド0。変更なし |
| **iii7** | Phrygian | **Dorian** | Phrygianのb2がアボイド。ドリアン一発処理 |
| **IV△7** | Lydian | Lydian | アボイド0。変更なし |
| **V7** | Mixolydian | Mixolydian | アボイド0。変更なし（テンション追加時は別途） |
| **vi7** | Aeolian | **Dorian** | Aeolianのb6がアボイド。ドリアン一発処理 |
| **viiø7** | Locrian | **Locrian #2** | Locrianのb2がアボイド。Mel.Min第6モード |

### なぜ「マイナーならドリアン一発」なのか

- ドリアンはアボイドノートを持たない → 9th, 13thが自由に使える
- 全マイナーコード(II, III, VI)をドリアン解釈 → II-Vフレーズのストックをそのまま流用可能
- フリジアンのb2やエオリアンのb6はコードトーンとぶつかりやすい

**深い動機: 調性からの解放**
- iii7をフリジアンで弾く → 「Cメジャーの3番目」というキーへの帰属意識が生まれる
- iii7をドリアンで弾く → そのコードだけの世界。**キーの重力から自由になる**
- つまり「マイナーならドリアン一発」は単に便利だからではなく、**キーの概念を希薄にし、調性音楽の重力から逃れるための手法**
- マイルスのモード奏法の本質がここにある
- さらに「あえて踏み外す」— アボイドを知った上でぶつける。**自由であるためにまず理論を知る、でも理論に従うためじゃない**（うりなみさん）
- **見た目は調性音楽でもアプローチはモーダル**: Stevie Wonderはコード進行が調性的に見えても、マイナーコードでドリアンモードとして独立した世界を作る。多くの分析は「Key=CのII-V-I」で止まるが、**演奏者がどのスケールでアプローチしているか**の層が読めていない。この分析ができる人は極めて少ない
- **コードネーム ≠ 演奏者の世界**: 記譜上は「Dm7」としか書けない。しかし演奏者の頭の中ではDドリアン全体が見えている。だからドリアン内のトライアド（Em, F, G, Am等）を自由に動かせる。コードの構成音に閉じた演奏と、スケール全体を見た演奏は全く別物（Upper Structure Triad的アプローチ）
- **64 Pad ExplorerのScale Overlay**: Dm7 + DorianのオーバーレイでDドリアン全体がパッド上に見える → トライアドの候補も視覚的に把握可能。この「コードの上のスケールの景色」を見せることが、HPSの教育コンテンツの核心

### R&B/ファンクにおける第2転回形トライアドとクォータル思考

**R&B、ファンク、ジャズ・ファンクでは第2転回形のトライアドが多用される。理由は理論書に書かれていないが、うりなみさんが理論化済み。**

**なぜ第2転回形か**:
- 第2転回形（例: C/G = G C E）は**ボトムにP4（完全4度）音程**を持つ
- P4 = クォータルサウンド = **調性音楽から離れる響き**
- 3度堆積（調性音楽の基盤）ではなく4度堆積で思考する → **モーダル/ファンクのアプローチと同じ構造**

**DドリアンにおけるF・G・Cトライアドの理由**:

| トライアド | 構成音 | Dm7との関係 | 役割 |
|-----------|--------|-----------|------|
| **F（bIII）** | F A C | b3, 5, b7 = **全部Dm7のコードトーン** | m7の「家」。コードサウンドそのもの |
| **G（IV）** | G B D | 11, **13(=ドリアン特性音)**, Root | ドリアンの「色」。Bがドリアンを確定させる |
| **C（bVII）** | C E G | b7, 9, 11 | エクステンション。テンション音で広がりを出す |

**二重の4度構造**:
- **ミクロ**: 各トライアドの第2転回形 → ボトムにP4（ヴォイシング内）
- **マクロ**: 3つのトライアドのルート G - C - F = **P4 - P4（4度堆積）**
- どのレベルで見ても4度。**徹底的に調性から離れている**

**ブルースとの共通構造**:
- ブルースでは**コードの4度上のトライアド**を弾く（例: C7の上にFトライアド）
- 調性音楽では11th（F）はC7に対して**アヴォイドノート**（3rdの半音上）
- しかしブルースは調性音楽ではない → C7(11)として正当化された
- **m7(13)と同じパターン**: 理論が実践に追いついた。先に音楽があって、後からコードネームが認めた
- **コードネームは調性音楽の記譜法。しかし演奏者は調性音楽をやっていない** — これがすべてに共通する本質

### m7(13) — モードを超えた標準テンション

**m7(13)のナチュラル13th = ドリアンの特性音（長6度）。もはやモード意識なしで普通に使う。**

| 根拠 | 説明 |
|---|---|
| **ダイアトニックの正当性** | II-V-IのIIm7（例: Key=CのDm7）はドリアン対応。13thはダイアトニック内の音を弾いているだけ |
| **暗さの回避** | エオリアン/フリジアンのb13は重い暗さ。ドリアンの13thは**都会的・洗練されたマイナー** |
| **ドミナントへの接続** | Dm7(13)のB音 = G7の3度(Guide Tone) = C△7の7度。先取りでスムーズな進行 |

**結論**: m7(13)は「ドリアンの手法」ではなく「マイナー7thを少しお洒落に響かせる常套手段」。機能和声の中で普通に使用される。

**トライトーン問題と許容**:
- m7(13)のb3と13thの間に**トライトーン（増4度）**が発生する（例: Cm7(13)のEbとA = F7の3度とb7度）
- **機能和声の立場**: ドミナント7thっぽく聞こえてサブドミナント機能が曖昧になる → 「避けるべき」（リハーモナイゼーション・テクニック）
- **モード/ファンクの立場**: そのピリッとしたエッジが**ファンキーな響き**として完全に許容される
- **現代の実践**: II-V進行のIIとしてスムーズに流したい時は邪魔になることもあるが、**ワンコードのファンクやモードジャズではむしろ積極使用**。都会的で泥臭い独特のグルーヴを作る
- **結論**: かつての「トライトーン=不安定=NG」から、現代では「ファンキーな響きとして許容」に進化した

**アプリへの影響**: V2.5のCategory Gテンションdimmingで、m7の13thはdimmingしていない（正しい挙動）。b9/#9/b13のみdimming。

### m7コードのテンション可能性（機能別3パターン）

**同じm7でも、キーの中の度数（機能/モード）でテンションが変わる。**

| 機能 | モード | 使えるテンション | アボイド | 理由 |
|---|---|---|---|---|
| **IIm7** | Dorian | **9, 11, 13** | なし | 最も自由。ファンク/ジャズで全部入り |
| **VIm7** | Aeolian | **9, 11** | b13 | b13(短6度)が5thと半音衝突 |
| **IIIm7** | Phrygian | **11のみ** | b9, b13 | b9(短9度)がRootと半音衝突、b13も5thと衝突 |

**共通で安全**: 9th, 11th（IIIm7のb9を除く）
**機能依存**: 13th（ドリアンのみ安全）、b13（全マイナーでアボイド傾向）

**出典**: 『実践コード・ワーク』『HOW TO IMPROVISE』

**#11はm7に不適切**: 標準的なm7スケール（Dorian, Aeolian, Phrygian）はすべてナチュラル4th。#11が現れるのはDorian #4（HM第4モード）のみで、実践ではほぼ使わない。一方△7の#11はリディアン＝ジャズ定番。

**アプリ実装**: Category Gでm7/m△7の#11をdim表示（V2.5+）。△7の#11はそのまま。Parent Scaleフィルタでも機能別に適用。

### モード感を出したい場合（あえてドリアン以外を選ぶ時）

| モード | 特性音 | サウンド | 使い方 |
|---|---|---|---|
| **Dorian** | 長6度(13th) | 都会的・モダン・少し明るいマイナー | Im6, Im13 |
| **Phrygian** | 短2度(b9) | スパニッシュ・ダーク | sus4(b9)（フリジアンコード）で使う |
| **Aeolian** | 短6度(b13) | ナチュラルマイナーの暗さ | 意図的にダークにしたい時 |

## 2. ドミナント7th系（解決先で2択）

### 基本原則

| 解決先 | テンション系統 | 第一候補スケール |
|---|---|---|
| **→ メジャーコード** | 9, 13（メジャー系） | **Mixolydian** |
| **→ マイナーコード** | b9, #9, b13（マイナー系） | **Altered or HMP5↓** |

### Dom7コードのテンション可能性

| テンション | 使えるか | 理由 |
|---|---|---|
| **9th** | **常用** | メジャー系。Mixolydian |
| **b9th** | **常用** | マイナー系。Altered/HMP5↓ |
| **#9th** | **常用** | マイナー系。Altered |
| **#11th** | **常用** | Lydian b7（裏コード） |
| **13th** | **常用** | メジャー系。Mixolydian |
| **b13th** | **常用** | マイナー系。Altered/HMP5↓ |
| **aug, b5** | **使える** | オルタードコンテキスト |
| **11th** | **アボイド（唯一）** | 3rd（長3度）と半音衝突。sus4にするなら3rdを省く |

**ドミナント7thは最もテンションが自由なコード。避けるのは11thだけ。**
sus4は3rdを置き換える別コードタイプ（7sus4）であり、11thテンションとは区別する。

**アプリ実装**: Category Gでdom7の11th(pc=5)を含むテンションを完全非表示（V2.5+）。sus4(replace3)はスルー。

### テンションは解決先を暗示する

- **13th**(E音 on G7) = C△7の3度 → **メジャー解決を暗示**
- **b13th**(Eb音 on G7) = Cm7のb3度 → **マイナー解決を暗示**
- **9th**(A音 on G7) = Cメジャースケール第6音 → メジャーキー暗示
- **b9th**(Ab音 on G7) = Cナチュラルマイナー第6音 → マイナーキー暗示

### セカンダリードミナント（Key=C）

| セカンダリーDom | 解決先 | 系統 | 第一候補 |
|---|---|---|---|
| C7 (I7) | → F△7 | メジャー | **Mixolydian** |
| D7 (II7) | → G7 | メジャー | **Mixolydian** |
| E7 (III7) | → Am7 | マイナー | **HMP5↓ or Altered** |
| A7 (VI7) | → Dm7 | マイナー | **Altered**（第一候補） |
| B7 (VII7) | → Em7 | マイナー | **HMP5↓ or Altered** |

### A7(VI7)が特に重要な例

- A Mixolydian(b6): 9th(メジャー系) + b13(マイナー系) が混在 → **曖昧なサウンド**
- 解決先がDm7(マイナー) → マイナー系テンション(b9, #9, b13)の方が自然
- **実践ではAltered or HMP5↓が第一候補**、Mixolydian(b6)は第二候補

### なぜAlteredがHMP5↓より優先されるか

| | HMP5↓ | Altered |
|---|---|---|
| b9, b13 | ある | ある |
| 4th(11th) | **アボイド** | ない |
| #9th | ない | **ある** |
| b5th(#11th) | ない | **ある** |
| **アボイド数** | **1** | **0** |

- Alteredはアボイドゼロで、さらに#9th, b5thも使える
- **#9thがコードに明記されていたら** → HMP5↓にはその音がない → **Alteredが自動的に第一候補**
- ただし実践では#9thはよく使うがb5th(#11th)はあまり使わない

### 裏コード（トライトーンサブスティテューション）

| コード | スケール | 理由 |
|---|---|---|
| **subV7（bII7）** | **Lydian b7** | Mel.Min第4モード |

- **Db Lydian b7 = G Altered（同じ7音）**。裏コードとオルタードは表裏一体
- #11(Db7のG音) = 元V7(G7)のルート → 裏コードがV7代理として機能する理由
- 「HOW TO IMPROVISE」のチャートでもbII7→Lydian b7が明記

### 7sus4（ハイブリッドコード）

| コード | スケール | 由来 |
|---|---|---|
| 7sus4 | **Mixolydian sus4** | Major第5モード（3rdをアボイド扱い、4thをコードトーン扱い） |
| 7sus4(b9) | **Mixolydian sus4 b9** | = HMP5↓と同構成音。フリジアンコード |

**なぜ分数コードでなく7sus4表記か**: 分数コード(Bb/C)では機能が見えない。7sus4ならドミナント機能が明確で、スケール選択に直結する。

## 3. マイナーキーのコード

### m7b5（ハーフディミニッシュ）の3層

| 層 | スケール | 由来 | 特徴 |
|---|---|---|---|
| ダイアトニック基本 | Locrian | Major第7モード | b2がアボイド |
| マイナーキー基本 | **Locrian #6** | Harm.Min第2モード | b2がアボイド、b6→#6 |
| **ジャズ実践** | **Locrian #2** | Mel.Min第6モード | **b2→ナチュラル9でアボイド解消** |

**実践的優先: Locrian #2 > Locrian #6 > Locrian**（アボイド少ない順）

※ バークリー式では Locrian #6 と呼ぶ（Locrian b13とは言わない）

### m7(b5)のテンション可能性

| テンション | 使えるか | 理由 |
|---|---|---|
| **11th** | **標準** | 全スケール（Locrian/Locrian #6/Locrian #2）で利用可能 |
| **b13th** | **標準** | Locrian/Locrian #6で利用可能。ダークな響き |
| **9th** | **条件付き** | Locrian #2でのみ利用可能（Locrianではb9=アボイド） |
| **b9th** | アボイド | Rootと半音衝突。Locrianの最大の弱点 |
| **#11th** | **存在しない** | pc=6はすでにb5（コードトーン）。#11とb5は同音だが、m7(b5)ではb5がコードを定義する音なので#11として解釈できない |

**重要**: #11はm7(b5)上で「テンション」として存在しえない。b5=F#がコードトーン（Root-m3-b5-b7の一部）であり、同じ音をテンションとして追加しても何も変わらない。

**アプリ実装**: Category B（no-op検出）で#11単体を非表示。Category B+C（重複検出、改良complexity計算）で(9)(#11)等の組み合わせも非表示（V2.5+）。

### マイナーキーのV7

**Altered > HMP5↓**（アボイド0 vs アボイド1）

### dim7（機能的ディミニッシュ）

| 選択肢 | スケール | 理由 |
|---|---|---|
| **実践第一** | **コンディミ**（Half-Whole Diminished） | アボイドなし、シンメトリック |
| 理論的 | HM第7モード（Functional Diminish） | = V7(b9)代理のHMP5↓を別ルートから |

- VIIdim7 = V7(b9)/3rd Bass（3度ベースの分数コード）
- よってV7(b9)上のHMP5↓フレーズがそのまま使える

### m6コード（文脈依存）

**同じCm6でも、何から変化したかでスケールが変わる。**

| 変化元 | スケール | 理由 |
|---|---|---|
| **m7 → m6** | **Dorian** | b7と6を両方含む。b7がアボイド |
| **m△7 → m6** | **Melodic Minor** | △7と6を両方含む |

- F7(9)/C（5度ベース）= Cm6 と同じ響き（等価関係）
- Cm6.9(Im6.9) → 演歌のサウンド → メロディックマイナーフレーズで締め

## 4. 3スケールシステムの実践的序列

### メジャースケール系（7モード）

全7モード実践で使う。ただしダイアトニック理論通りの割り当てではなく、アボイド回避でモードを差し替える（上記参照）。

### メロディックマイナー系（7モード）

| # | モード | コード | 実践 | 備考 |
|---|---|---|---|---|
| 1 | **Melodic Minor** ★ | Im△7 | 多用 | Im6(m△7由来)にも |
| 2 | Dorian b2 | IIm7 | 場面あり | |
| 3 | Lydian #5 | bIII△7(#5) | 場面あり | |
| 4 | **Lydian b7** ★ | IV7 | 多用 | 裏コード第一選択 |
| 5 | Mixolydian b6 | V7 | 第二候補 | A7(VI7)等で曖昧になりがち |
| 6 | **Locrian #2** ★ | VIm7(b5) | 多用 | m7b5のジャズ実践第一選択 |
| 7 | **Altered** ★ | VII7(b5) | 多用 | V7・セカンダリーDomで連発 |

**★4つが実戦の主力。**

### ハーモニックマイナー系（7モード）

| # | モード | コード | 実践 | 備考 |
|---|---|---|---|---|
| 1 | **Harmonic Minor** ★ | Im△7 | 使う | マイナーキーの基本 |
| 2 | Locrian #6 | IIm7(b5) | 理論的 | Locrian #2に劣後 |
| 3 | Ionian #5 | bIII△7(#5) | 理論的 | 実践的でない |
| 4 | Dorian #4 | IVm7 | 理論的 | F Dorian（ナチュラル）が普通 |
| 5 | **HMP5↓ / Phrygian Dominant** ★ | V7 | 多用 | マイナーキーV7の定番 |
| 6 | Lydian #2 | bVI△7 | 理論的 | |
| 7 | **Functional Diminish** ★ | VIIdim7 | 使う | = V7(b9)代理。日本語: パッシングDim |

**★はHMP5↓とFunctional Diminishだけ。他は理論的構築物。**
「スケール候補が複数ある時は、より自然な方が第一候補」（教科書の原文）

### 呼称の注意

| バークリー式 | 日本での通称 | うりなみさん |
|---|---|---|
| HMP5↓ | HMP5↓（唯一定着した名称） | Phrygian Dominant（機能が覚えやすい） |
| Functional Diminish | パッシングDim / V7(b9)代理 | — |
| 他のHMモード | 固有名なし | — |

## 5. テンション命名規則（add9 vs 9）

### 原則: 7thの有無でテンション表記が変わる

| 7thの有無 | 9thの追加 | 正しい表記 | 構成音 | 誤った表記 |
|---|---|---|---|---|
| **7thあり** | +9th | **Cmaj9** | C E G B D | ~~Cmaj7(add9)~~ |
| **7thなし** | +9th | **Cadd9** | C E G D | ~~C9~~ |

### なぜこの区別が必要か

- **「9」は7thを含意する**: C9 = C E G Bb D（ドミナント9th）。7thが暗黙的に含まれる
- **「add」は7thを省く**: Cadd9 = C E G D。「トライアドに9thだけを足す」の意味
- **7th + 9thなら「9thコード」に格上げ**: Cmaj7 + 9th = Cmaj9。わざわざ「add」を付ける必要がない

### 出典

- 『実践コード・ワーク』: 「add9のコードは通常、メジャー7thの音をフォローしないものがほとんど」
- 『ジャズ・ギター リズム＆バックグラウンド・コード』: IタイプコードとしてImaj7と並んでImaj9を定義

### 同様のパターン（11th, 13th）

| コード | 構成音 | 意味 |
|---|---|---|
| C11 | C E G Bb D F | 7th + 9th + 11th（全部入り） |
| Cadd11 | C E G F | トライアド + 11thのみ |
| C13 | C E G Bb D A | 7th + 9th + 13th |

**「add」= 7thなし。数字だけ = 7th（と間の度数）を含意。**

### 6thコードの特例

6thコード（C6, Cm6）は4音コードだが7thを含まない。テンションの扱いが独特:

| テンション | メジャー6th | マイナー6th | 理由 |
|---|---|---|---|
| **9th** | **使える** | **使える** | C6+9th = 「C6/9」。「add9」とは呼ばない |
| **11th** | **アボイド** | **使える** | メジャー6thは3rd(pc=4)と11th(pc=5)が半音衝突。m6はm3(pc=3)と全音離れてOK |
| **#11th** | 使える | 使える | リディアン文脈 |
| **sus4** | **なし** | **なし** | 3rdが6thコードの本質。sus4は3rdを消す→構造的矛盾 |
| **add9** | **なし** | **なし** | 6thコードでは「add9」ではなく「9」と呼ぶ |

**命名**: C6 + 9th = **C6/9**（シックス・ナイン）。「C6(add9)」は誤り。6thコードは7thと同じ拡張体系に属するため、テンション名は数字のみ。

**出典**: 『ジャズ・ギター リズム＆バックグラウンド・コード』: 「メジャー6/9コードは、メジャー6thコードに長9度を加えることによって得られる」

**アプリ実装**: Category Dで6thコードのsus4を非表示。Category G2でメジャー6thの11th(pc=5)を非表示。Category Hでhas6th時にadd9非表示・9表示（V2.5+）。

### Maj7コードのテンション可能性

| テンション | 使えるか | 理由 |
|---|---|---|
| **9th** | **常用** | 最頻出。機能を阻害しない |
| **13th** | **使える** | 7thと隣接時は配慮必要。6thコードとして7th省略も多い |
| **#11th** | **ジャズ定番** | リディアンの響き。Imaj7でも浮遊感・モダンさのため定石化 |
| **11th** | **アボイド** | 3rd（長3度）と半音でぶつかる。sus4にするなら別だがMaj7ではない |

**Imaj7 vs IVmaj7の#11th**:
- IVmaj7: ダイアトニックでナチュラルに#11th発生（リディアン）
- Imaj7: 本来はアイオニアン（#11なし）だが、完全終止の明確さを抑えるためあえてリディアンを使う

**出典**: 『実践コード・ワーク』『ジャズ・ギター リズム＆バックグラウンド・コード』『HOW TO IMPROVISE』

### b5/#5 vs #11/b13 — テンションかコードトーン変化か

**同じ音でも、解釈で名前が変わる。**

| 音 | テンション名 | コードトーン変化名 | 違い |
|---|---|---|---|
| 5thの半音下 | **#11** | **b5** | #11=5thと共存（テンション付加）、b5=5thを置換 |
| 5thの半音上 | **b13** | **#5 (aug)** | b13=5thと共存（テンション付加）、#5=5thを置換 |

**使い分けの傾向**:
- **#11**: Lydian b7由来。5thと共存可能（理論上）。裏コード等
- **b5**: Altered/Whole Tone由来。5thが存在しないスケール。5thの置換
- **b13**: Altered/HMP5↓由来。解決先の9thへ半音下行する性質
- **#5 (aug)**: Whole Tone由来。クリシェライン(5→#5→6)で使用

**現代ジャズ理論**: テンション名（#11, b13）で呼ぶ方が一般的。コード・スケールとの関連が見えやすい。
**リードシート**: 視覚的に短く書くためb5, aug表記が残る（G7(b5), G7aug等）。

**出典**: 『実践コード・ワーク』（テンション分類）、『ザ・ジャズ・セオリー』（異名同音の互換性に言及）

**アプリ実装**: `getBuilderChordName()`で7thコード文脈のb5→#11変換を実装済み（V2.2+）。UIでは両方をボタンとして提供（b5=質の変更、#11=テンション付加として区別）。

### アプリ実装（Category G+H）

64 Pad Explorer V2.5+で実装:
- **Category G**: dom7で11th(pc=5)を完全非表示。m7/m△7で#11(pc=6)をdim
- **Category H**: has7th → `add9`非表示、!has7th → `9`非表示

## 6. 実装への示唆（64 Pad Explorer）

### ソートロジック改善 — V2.4実装済み（2026-02-17）

**修正前のEm7 (iii7) in C の自動選択**: E Phrygian（avoid=2, distance=0）
**修正後**: E Dorian（avoid=0, distance=2）

**Practical / Diatonic トグル実装**:
- **Practical**: avoidCount → distance → system → degreeNum + DIATONIC_AUTO_PREF
- **Diatonic**: distance → system → degreeNum（V2.3以前の動作）

**DIATONIC_AUTO_PREF（ダイアトニック度数→推奨scaleIdx）**:
```
I△7  → Lydian (3), Ionian (0)
ii7  → Dorian (1)
iii7 → Dorian (1)
IV△7 → Lydian (3), Ionian (0)
V7   → Mixolydian (4), Lydian b7 (17)
vi7  → Dorian (1), Aeolian (5)
viiø7→ Locrian ♮2 (19), Locrian (6)
```

**データ修正（SCALE_AVAIL_TENSIONS）**:
- Phrygian: `avoid:['b9']` → `avoid:['b9','b13']`（b13=pc8は5thの半音上）
- Aeolian: avoidなし → `avoid:['b13']`（同上）

### 文脈が必要なケース

| ケース | 必要な情報 | 現状のアプリ |
|---|---|---|
| セカンダリーDomの解決先 | 次のコード | **持っていない**（単一コード逆引き） |
| m6の変化元(m7 or m△7) | 前のコード | **持っていない** |
| viiø7のキー文脈（メジャー/マイナー） | キー情報 | **持っている**（ダイアトニックバーから推定可能） |

### 文脈なしでも改善できること

1. **アボイド数ソート**: 全ケースで即座に改善できる
2. **ダイアトニック度数からの推定**: iii7→Dorian, vi7→Dorian等
3. **#9thテンション検出**: #9th付き7th → Altered優先
4. **裏コード検出**: ルートとキーの関係（b2=bII）→ Lydian b7優先

### 将来的に文脈を入れる場合

メモリースロットにコード進行が入っている → 前後のコードから解決先を推定可能。ただしこれはSequenceモード（PAD DAW）のスコープ。


## コード・スケールチャート

資料『HOW TO COMP』および『HOW TO IMPROVISE』の「コード・スケール・チャート」を中心に、これまでの会話で話題になったポイント（テンションの可否など）を加味して、コード・タイプと対応するスケールの関係をまとめました。

ジャズのアドリブやコンピング（伴奏）において、最も基本的かつ実践的な分類です。

### 1. メジャー系 (Major)

| コード・タイプ | 対応スケール | 構成音・特徴 |
| :--- | :--- | :--- |
| **Maj7 / 6** | **アイオニアン** (Ionian) | メジャー・スケールそのもの。Imaj7（トニック）で使用。 |
| | **リディアン** (Lydian) | **#11** を含む。IVmaj7（サブドミナント）や、モダンなトニックで使用。 |
| **Maj7(#5)** | **リディアン・オーギュメント**<br>(Lydian Augmented) | メロディック・マイナーの第3モード。**#5** と **#11** を含む。 |
| | **アイオニアン・オーギュメント**<br>(Ionian Augmented) | ハーモニック・マイナーの第3モード。**#5** を含む。 |

### 2. ドミナント系 (Dominant 7th)

最も選択肢が多く、機能（どこへ解決するか）によって使い分けます。

| コード・タイプ | 対応スケール | 構成音・特徴 |
| :--- | :--- | :--- |
| **7 (Normal)** | **ミクソリディアン**<br>(Mixolydian) | 基本形。**9, 13** を含む。I（メジャー）へ解決するV7で使用。 |
| **7(#11) / 7(b5)** | **リディアン b7**<br>(Lydian b7) | メロディック・マイナーの第4モード。**9, #11, 13** を含む。裏コード（bII7）やIV7で使用。 |
| **7(b9, #9, alt)** | **オルタード**<br>(Altered / Super Locrian) | **b9, #9, b5(#11), b13** を含む。全てのテンションが変化している。マイナーやメジャーへの強い解決感。 |
| **7(b9, 13)** | **コンディミ**<br>(Half-Whole Diminished) | **b9, #9, #11, 13** を含む。**b9とナチュラル13が共存**する唯一のスケール。 |
| **7(#5)** | **ホールトーン**<br>(Whole Tone) | 全音音階。**9, #5, #11** を含む。独特の浮遊感。 |
| **7(b9, b13)** | **HMP5↓**<br>(Mixolydian b9, b13) | ハーモニック・マイナーの第5モード。**b9, b13** を含む。マイナー・キーのV7で多用（完全なマイナー・ドミナント）。 |
| **7sus4** | **ミクソリディアン sus4** | 3度の代わりに4度を使用。アボイドなし。 |

### 3. マイナー系 (Minor)

| コード・タイプ | 対応スケール | 構成音・特徴 |
| :--- | :--- | :--- |
| **m7** | **ドリアン** (Dorian) | **11, 13** を含む。II-VのIIm7で最も一般的。 |
| | **エオリアン** (Aeolian) | **b13** を含む。VIm7やIm7（ナチュラル・マイナー）で使用。 |
| | **フリジアン** (Phrygian) | **b9, b13** を含む。IIIm7で使用。スパニッシュな響き。 |
| **m6 / mM7** | **メロディック・マイナー** | 上行形。**9, 11, 13, Maj7** を含む。トニック・マイナー（Im6, ImM7）で使用。 |
| | **ハーモニック・マイナー** | **b6(b13)** を含む。ImM7で使用されるが、メロディック・マイナーより頻度は低い。 |

### 4. ハーフ・ディミニッシュ / ディミニッシュ (Half-Dim / Dim)

| コード・タイプ | 対応スケール | 構成音・特徴 |
| :--- | :--- | :--- |
| **m7(b5)** | **ロクリアン**<br>(Locrian) | **b9, 11, b13** を含む。メジャー・キーのVIIm7(b5)などで使用。 |
| | **ロクリアン #2**<br>(Locrian #2 / #9) | **Natural 9, 11** を含む。メロディック・マイナーの第6モード。**9th**を使いたい時（マイナー・キーのIIm7(b5)など）に多用される。 |
| **dim7** | **ディミニッシュ**<br>(Whole-Half Diminished) | 全音-半音の配列。**Maj7**を含む8音スケール。パッシング・ディミニッシュなどで使用。 |

### 補足：スケール選択のヒント（資料より）
*   **アボイド・ノート**: スケール・トーンの中で、コード・トーン（特に3度）と半音でぶつかる音（例：アイオニアンの4度、ミクソリディアンの4度）は、長く伸ばすと響きを損なうため注意が必要です。
*   **実践的な運用**: 多くのジャズ・プレイヤーは、これら全てのスケールを丸暗記するのではなく、**「メジャー」「メロディック・マイナー」「ディミニッシュ」「ホールトーン」の4つの親スケール**から派生するものとして整理しています。

## ペンタトニック / ブルーススケールの位置づけ（2026-02-18 壁打ち）

### なぜAvailable Scaleに入れないか

- **ペンタトニックはモード**。キー全体に対して横に敷くもの。コード・スケール理論（縦の対応: コード→スケール）とは体系が異なる
- 事実上、全てのコードタイプでペンタトニックは使える。「このコードにはこのペンタ」という1対1対応にならない
- ペンタトニック・ハーモニーは4度堆積の世界。64 Pad Explorerは3度堆積のコード・スケール理論が土台

### 実戦での重要度

- **ペンタ/ブルーススケールは軽んじられがちだが、使いこなすのは極めて難しい**。ブルースのプレイヤーが抜群に上手い
- 音数が少ないからこそリズムとニュアンスがむき出しになる。ごまかしが効かない
- ソウル・ファンクでは中途半端にスケール（ドリアン等）を使うと「らしさ」を失う
- トライアド/四和音+クロマチック、メジャー/マイナーペンタ、ブルース、ソウルスケールが基本中の基本
- **実戦の手段としては第一に来るべきもの**（うりなみさんの見解）

### 落としどころ

- 64 Pad Explorer（縦のコード・スケール理論）には入れない
- **リズム譜アプリ**（横の流れ）でスケール/アプローチ提案機能として実装する
- 進行全体を見て「ここはペンタで通せる」「ここはトライアド+クロマチック」と提案できる形
